# Rent Roll — Collections View

> **Date:** 2026-07-05
> **Goal:** Let the owner see which tenants owe rent for the current period, with balance, status, and property/unit info.

---

## Architecture

1. **Backend**: New `GET /payments/rent-roll` endpoint that joins Lease → Tenant, Unit, Property, and RentSchedule to build a per-tenant collections view for the current billing period
2. **Frontend**: Add a "Rent Roll" tab to the Payments page showing a table of tenants with their rent status, balance, and property info

---

## Data Flow

```
RentSchedule (current period)
  → Lease (active)
    → Tenant (name, email, phone)
    → Unit (unit_code, rent_amount)
      → Property (name, location)
```

---

## Tasks

### Task 1: Backend — Rent Roll endpoint

Create `GET /payments/rent-roll` in payments.py:
- Query active leases with their current-period rent schedule
- Join through to tenant, unit, property
- Filter by assigned properties for non-owner roles
- Return: [{tenant_name, tenant_email, unit_code, property_name, monthly_rent, expected_amount, paid_amount, balance, status, due_date}]

### Task 2: Frontend — Rent Roll tab

Add "Rent Roll" as a sub-tab or section in LandlordPayments.tsx:
- Fetch from `/payments/rent-roll`
- Table columns: Tenant, Unit, Property, Rent Due, Paid, Balance, Status, Due Date
- Status badges: Paid (green), Partial (amber), Pending (red), Overdue (red pulsing)
- Summary card at top: total expected, total collected, total outstanding, # delinquent
- Filter by status (All/Pending/Paid/Overdue)
